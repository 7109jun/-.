## Joke가 무엇인가요?

Joke는 데이터를 표현하기 위한 텍스트 포맷입니다.
JSON처럼 기계가 파싱할 수 있으면서도, 사람이 눈으로 읽었을 때 바로 이해할 수 있도록 설계되었습니다.

핵심 철학은 세 가지입니다.

1. 읽기 쉬울 것
2. 쓸 때 짧을 것
3. 수식을 데이터 안에 그대로 넣을 수 있을 것

이 문서에서는 Joke의 모든 문법을 처음부터 끝까지 설명합니다.

---

## 1. 문자열

문자열은 소괄호 `( )`로 감쌉니다.

    (hello)
    (hello world)
    (안녕하세요)
    (こんにちは)
    (🎉🚀)

공백, 한글, 일본어, 이모지 모두 그대로 사용할 수 있습니다.

### 특수문자 이스케이프

문자열 안에 소괄호나 줄바꿈을 넣고 싶으면 백슬래시 `\`를 사용합니다.

|写法|의미|
|---|---|
|`\\`|백슬래시 자체|
|`\(`|왼쪽 소괄호|
|`\)`|오른쪽 소괄호|
|`\n`|줄바꿈|
|`\r`|캐리지 리턴|
|`\t`|탭|

예시:

    (hello \(world\))

이것은 `hello (world)` 라는 문자열입니다.

---

## 2. 원시 값 (Primitive)

숫자, 불리언, null은 `<() >`로 감쌉니다.

|종류|Joke|JSON으로 변환 결과|
|---|---|---|
|정수|`<()>100>`|`100`|
|실수|`<()>3.14>`|`3.14`|
|참|`<()>true>`|`true`|
|거짓|`<()>false>`|`false`|
|널|`<()>null>`|`null`|

왜 `<() >`인가요?
`<`는 "값이 시작된다", `()`는 "내용을 감싼다", `>`는 "값이 끝난다"는 의미입니다.
문자열 `(hello)`과 숫자 `100`을 구분하기 위해 이 구조를 사용합니다.

---

## 3. 배열

배열은 `<[]>`로 시작하고 `>`로 끝냅니다.
각 요소는 새 줄에 씁니다.

    <[]>
        (A)
        (B)
        <()100>
    >

JSON으로 변환하면:

    ["A", "B", 100]

### 빈 배열

    <[]>
    >

JSON:

    []

### 중첩 배열

    <[]>
        <[]>
            <()1>
            <()2>
        >
        <[]>
            <()3>
            <()4>
        >
    >

JSON:

    [[1, 2], [3, 4]]

---

## 4. 객체

객체는 중괄호 `{ }`로 감쌉니다.
각 속성은 `(키) = 값` 형태로 씁니다.

    {
        (name) = (Ruru)
        (age) = <()13>
        (active) = <()true>
    }

JSON으로 변환하면:

    {
      "name": "Ruru",
      "age": 13,
      "active": true
    }

### 빈 객체

    {
    }

JSON:

    {}

### 중첩 객체

    {
        (user) = {
            (name) = (Ruru)
            (age) = <()13>
        }
        (scores) = <[]>
            <()95>
            <()87>
        >
    }

JSON:

    {
      "user": {
        "name": "Ruru",
        "age": 13
      },
      "scores": [95, 87]
    }

### 객체와 수식의 구분 규칙

Joke에서 중괄호 `{ }`는 두 가지 용도로 쓰입니다.

- **객체**: `{` 뒤에 `(키) =` 패턴이 옴
- **수식**: `{` 뒤에 숫자 또는 `{`가 옴

파서는 이 규칙으로 두 가지를 자동으로 구분합니다.

---

## 5. 주석

`#`으로 시작하는 줄은 무시됩니다.

    # 이것은 주석입니다
    (name) = (Ruru)
    # 아래도 주석입니다
    (age) = <()13>

인라인 주석(줄 중간에 `#`)은 지원하지 않습니다.

---

## 6. 수식 (Expression)

Joke는 데이터 안에 수식을 직접 쓸 수 있습니다.
수식은 중괄호 `{ }`와 연산자로 구성됩니다.

### 기본 수식

    {5}(+){3}

결과: `8`

### 중첩 수식

    {{5}(+){3}}(x){2}

계산 순서:
1. 안쪽: `{5}(+){3}` → `8`
2. 바깥쪽: `{8}(x){2}` → `16`

결과: `16`

### 연산자 목록

|연산자|의미|예시|결과|
|---|---|---|---|
|`(+)`|덧셈|`{5}(+){3}`|`8`|
|`(-)`|뺄셈|`{10}(-){4}`|`6`|
|`(x)`|곱셈|`{3}(x){7}`|`21`|
|`(/)`|나눗셈|`{20}(/){4}`|`5`|

### 연산자 우선순위

1. 중괄호 `{ }`로 감싼 부분 (가장 먼저 계산)
2. 곱셈 `(x)`
3. 나눗셈 `(/)`
4. 덧셈 `(+)`
5. 뺄셈 `(-)`

### 수식을 객체 안에서 사용하기

    {
        (width) = <()10>
        (height) = <()5>
        (area) = {10}(x){5}
        (total) = {{10}(x){5}}(+){3}
    }

JSON:

    {
      "width": 10,
      "height": 5,
      "area": 50,
      "total": 53
    }

---

## 7. 참조 (Reference)

`*`로 시작하면 참조입니다.
다른 곳에서 정의된 값을 가리킬 때 사용합니다.

    *abc
    *config.database.host
    *global.settings

JSON으로 변환하면 문자열로 유지됩니다.

    "*abc"
    "*config.database.host"

---

## 8. 경로 (Path)

`/`로 시작하면 경로입니다.
설정 파일의 위치나 계층 구조를 나타낼 때 사용합니다.

    /config/user
    /app/settings/theme
    /data/logs/2026

JSON으로 변환하면 문자열로 유지됩니다.

    "/config/user"

---

## 9. 유니코드 지원

Joke는 UTF-8 기반입니다. 아래가 모두 동작합니다.

- 한글: `(안녕하세요)`
- 일본어: `(こんにちは)`
- 중국어: `(你好)`
- 이모지: `(🎉🚀✨)`
- 혼합: `(Hello 세계 🌍)`

---

## 10. 왕복 변환 보장

Joke 엔진은 양방향 변환을 지원합니다.

    JSON → Joke → JSON  (원본과 동일)
    Joke → JSON → Joke  (원본과 동일)

데이터 손실 없이 언제든 변환할 수 있습니다.

## 11. 전체 예제

### Joke

    # 사용자 설정 파일
    {
        (name) = (Ruru)
        (age) = <()13>
        (active) = <()true>
        (score) = {{5}(+){3}}(x){2}

        "city": "Seoul",

      },

    }

## 12. 문법 요약 (EBNF)

    joke       = value ;
    value      = string | primitive | array | object
               | expression | reference | path ;

    string     = "(" , { 문자 } , ")" ;
    primitive  = "<()" , ( 숫자 | "true" | "false" | "null" ) , ">" ;

    array      = "<[]>" , { value } , ">" ;
    object     = "{" , { property } , "}" ;
    property   = "(" , 키 , ")" , "=" , value ;

    expression = "{" , 항 , "}" , 연산자 , "{" , 항 , "}" ;
    항         = 숫자 | expression ;
    연산자     = "(+)" | "(-)" | "(x)" | "(/)" ;

    reference  = "*" , 식별자 , { "." , 식별자 } ;
    path       = "/" , 세그먼트 , { "/" , 세그먼트 } ;

    comment    = "#" , { 문자 } ;

## 13. 파서 규칙

- Recursive Descent Parser 사용
- eval, Function() 사용 금지
- 정규식으로 수식 계산 금지
- 수식은 전용 인터프리터로 계산
- 오류 발생 시 Line, Column, Expected, Found 출력
